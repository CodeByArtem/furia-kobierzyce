# Правила разработки проекта "Furia Kobierzyce"

Ты — эксперт-разработчик, работающий над проектом Furia Kobierzyce. Следуй этим правилам:

## 1. Стек технологий
- **Frontend**: Next.js 15+ (App Router), React, TypeScript.
- **Стили**: Только SCSS (CSS Modules). **Никакого Tailwind CSS**.
- **Язык**: TypeScript. Строгая типизация всех пропсов компонентов.

## 2. Архитектура стилей
- Все стили пиши в файлах `.module.scss`.
- Импортируй их как `import styles from './ComponentName.module.scss'`.
- Используй переменные из `src/styles/variables.scss` для цветов.
- Избегай глобальных классов, если это не сброс стилей в `globals.scss`.

## 3. Структура компонентов
- Каждый компонент — в отдельной папке: `src/components/ComponentName/`.
- В папке всегда два файла: `ComponentName.tsx` и `ComponentName.module.scss`.
- Используй функциональные компоненты и стрелочные функции.

## 4. Качество кода
- Код должен быть чистым, семантичным (используй <header>, <main>, <section>, <footer>).
- Имя файлов компонентов — PascalCase.
- Всегда используй `@` алиас для импортов (например, `import { Button } from '@/components/Button'`).
- Если нужен интерактив, добавляй `'use client'` в начале файла.

## 5. Общение
- Если не уверен в структуре данных, предложи её, основываясь на задачах проекта.
- Пиши лаконичный код, не добавляй лишних комментариев, если логика очевидна.
